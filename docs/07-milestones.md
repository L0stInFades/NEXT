# 07 里程碑：从原型到“两宋首作可交付”

> 目标：用“可验证的演示切片”推动自研引擎落地，避免长期不可见的技术堆叠。

## 7.1 总体节奏：先可跑、再可做、再可规模化

推荐把里程碑拆成三条主线并行推进：
- **Core 主线**：Foundation/Runtime/Asset/Job/可观测性（任何时候不崩）
- **World 主线**：分区流式/LOD/HLOD/无加载（两宋大地图的生命线）
- **Gameplay 主线**：手感/镜头/任务脚本（决定体验与差异化）

渲染与 AI 以“双轨”推进：
- Production：保证能做内容、能交付、能跑稳定
- R&D：每次只押注少量实验，且必须可回滚

## 7.2 Milestone 列表（建议）

### M0：技术基建与工程护栏（最先做，越早越好）
交付物：
- 可编译工程骨架（Platform/Foundation/Runtime 分层）
- 日志/Profiler/统计面板（最简版也要有）
- 资源编译器雏形（能把少量资源编译成运行时格式）

验收口径：
- 任意开发机一键构建并运行“空场景 + 相机移动”，可输出性能与内存统计。

### M1：手感与运镜原型（差异化的第一刀）
交付物：
- Camera Graph 骨架（第一/第三/锁定/轨道的统一切换）
- 角色控制器原型（输入响应曲线、阻尼、惯性、碰撞）

验收口径：
- 10 分钟持续移动/切换镜头不抖、不穿模、参数可调且可保存预设。

### M2：渲染垂直切片（Production Raster 基线）
交付物：
- RHI（DX12）+ Render Graph
- 基础 PBR 材质、阴影、后处理、调试视图

验收口径：
- 固定场景稳定输出；Render Pass 耗时可见；可做 Golden Image 对比。

### M3：世界分区与无加载切片（最关键风险点）
交付物：
- World Partition（Grid Cell）+ 资源流式加载（IO/解压/GPU 上传）
- LOD/HLOD 第一版（至少能让远景稳定）
- 相机轨迹预测驱动预取（为一镜到底准备）

验收口径：
- 固定轨迹跑 5–10 分钟无读条；加载尖峰可定位且可优化。

### M4：任务系统切片（“不断支线”的可验证雏形）
交付物：
- Event Bus + 基础声明式条件
- 任务图编辑器最小版（或数据格式 + 可视化）
- 一个“灾害扰动”用例：灾害发生后任务能改道/延后/转补救任务

验收口径：
- 同一任务在 2–3 种世界状态下都能完整闭环，不出现硬掐支线。

### M5：两宋生产工具链可用（内容吞吐开始爬坡）
交付物：
- 编辑器：世界编辑/分区预览/流式可视化
- 资源验证器：预算检查、时代约束检查（先从少数规则开始）
- Mod 打包雏形：manifest + 内容包 + 脚本沙盒（最小可运行）

验收口径：
- 设计师/美术能独立产出一个小区域并跑通任务闭环；Mod 能被加载与卸载。

### M6：光照分层完善（Tier B/C 稳定，Tier A 逐步落地）
交付物（Production）：
- Probe/DDGI + 屏幕空间补全（稳定优先）
- 反射的三层组合（SSR + Probes + 可选 RT）

交付物（R&D，可选）：
- 照片/电影模式 Path Tracing（限定场景/限定镜头）

验收口径：
- 同一镜头在 Tier B/C 画质稳定且性能达标；Tier A 可用于宣传/照片模式而不拖垮开发主干。

### M7：AI/生成进入生产（先离线，后运行时）
交付物：
- NPC 外观变体生成器（离线）+ 时代约束验证
- 建筑/街区形状语法工具链（输出可编辑中间表示）
- 植被/农田随时间变化的参数化系统

验收口径：
- 生成结果可被审核与打包；运行时不依赖在线生成；性能与预算可控。

## 7.3 每个里程碑的“硬门槛”（建议写进 CI/验收表）

- 性能：固定轨迹性能回归不允许退化超过阈值（阈值由团队设定）
- 稳定性：长时间跑（无加载 + 时间推进）无崩溃、无明显泄漏
- 可观测：新系统必须有统计/调试视图，否则不算完成
- 可回滚：实验功能没有 fallback 不允许进主干
