# 00 愿景、边界与原则

## 0.1 背景与愿景

目标是开发“中华历史朝代系列 IP”，用故事把每个朝代的大事与小事融合到同一套可交互的大世界里。首作设定在两宋，倾向**超大地图、无加载体验、一镜到底式叙事与大场面刻画**。

本引擎选择自研的核心动机（按优先级归纳）：
- **独特操作手感**：角色/镜头/输入响应的一致性与可调性比“默认模板”更重要。
- **影视级运镜与一镜到底**：统一一/三人称与电影镜头语言，形成可复用的镜头系统与制作管线。
- **顶级优化且可分层扩展画质**：保留路径追踪/全局光照能力，同时在老硬件上用改进的非光追手段逼近光追观感。
- **AI/生成驱动的生产力与世界复杂度**：可生成差异化 NPC、符合时代背景的建筑/街区/器物，并能服务布料/肌肉/骨骼/植物生长等高级系统。
- **超大世界加载与百年时间线**：高端压缩/流式加载、强 LOD/HLOD，支持长时间线的世界变化（“百年变迁”）。
- **更先进的逻辑脚本/任务系统**：面向复杂世界与随机灾害扰动，不以“传统关卡脚本”的方式硬掐支线。
- **可持续实验体系 + 可重构路径 + 高质量 Mod 生态**：实验不崩盘、重构有路可走；Mod 作者可以做出接近官方质量的任务/地图/资产，并能与主线系统建立联系、产生影响。

## 0.2 目标（必须）

### 体验目标
- 一/三人称切换与镜头语言一致；支持“手感参数化”（响应曲线、惯性、锁定、缓动、碰撞、肩膀切换、镜头阻尼等）。
- 运镜系统可覆盖：跟随、锁定、自由视角、过肩、骑乘、室内狭窄空间、电影轨道镜头、预设镜头段落、实时镜头调度。
- 大场面（人群/军阵/城市）稳定：近景高细节，远景通过 HLOD/实例化/群体模拟保持规模感。

### 技术目标
- 渲染质量分层（同一套资产与内容）：
  - **Tier A（旗舰）**：硬件 RT + 路径追踪（可选“电影模式”）。
  - **Tier B（主流）**：混合 RT（反射/阴影/局部 GI）+ 非 RT GI（探针/表面缓存/屏幕空间/体素或 SDF）。
  - **Tier C（老硬件）**：无 RT 的高质量 GI/反射替代方案（探针 + 屏幕空间 + HLOD/环境缓存），依然保持可接受观感与稳定帧率。
- 超大世界：分区流式加载、强一致的资源虚拟化（几何/纹理/动画/音频/脚本都可流式）。
- 百年时间线：世界变化可表达、可回放、可裁剪；存档与任务系统对“时间变化/灾害”具备韧性。

### 生产与长期演进目标
- “Production Track”保证可交付；“R&D Track”在不污染主干的前提下持续尝试新技术（材质/渲染/生成/压缩等）。
- 明确模块边界与可替换点，避免屎山：核心稳定、外层可重构。
- 可扩展 Mod：稳定 API、版本化资源与脚本、可控的安全边界与性能预算。

## 0.3 非目标（第一阶段明确不做/少做）

为避免自研引擎无限膨胀，必须设定第一阶段边界：
- 不追求“全平台同步起步”：建议先以 **Windows + DX12** 为主（渲染后端可预留 Vulkan/主机接口，但不强行同时落地）。
- 不从 0 重做 DCC：资产制作依赖现成工具（Blender/Maya/Substance 等），自研的是**导入/编译/验证/增量构建**与编辑器工作流。
- 不把“AI 生成”直接放进运行时硬依赖：首发版本以**离线生成 + 约束驱动的程序化生成**为主；运行时生成做为可选/可降级模块。
- 不一次性实现“全套影视级 PT”：路径追踪作为高端档与研发能力，先以可控范围（室内、特定镜头、照片模式/电影模式）落地。

## 0.4 指标与质量门槛（建议的验收口径）

这些指标用于“工程决策”，不用于对外宣传；后续可按团队资源调整。

### 性能与稳定性
- 帧率目标：Tier B/C 场景下优先保证 **稳定帧时间**（例如 16.6ms / 33.3ms 目标之一）。
- 帧波动控制：关键系统必须给出预算（CPU 主线程、Job、GPU、IO、内存、VRAM）。
- IO：任何“加载卡顿”视为 Bug；流式加载必须可观测（可视化流量、队列、命中率）。

### 可维护性
- 模块化：引擎核心（Core/Foundation）不得依赖游戏层（Game）。
- 数据版本化：资源格式与脚本接口必须可迁移（schema/versioning），否则 Mod 与长期项目会崩。
- 可回放：至少在开发期具备事件/任务/世界状态的可回放与录制能力（用于定位“复杂世界 bug”）。

## 0.5 架构原则（强约束）

1. **分层明确、依赖单向**：Platform → Foundation → Runtime → Feature（Renderer/World/AI/Gameplay）→ Tools → Game。
2. **数据驱动优先**：任务、世界事件、生成规则、镜头段落、AI 行为尽量用数据描述；代码实现“执行器”而不是堆 if/else。
3. **异步与预算化**：任何重活（IO/解压/编译/生成/烘焙）必须异步，并有预算与降级路径。
4. **可观测性是功能的一部分**：Profiler、日志、统计、可视化面板要在系统设计时预留。
5. **Production/R&D 双轨**：实验模块必须可被开关、隔离与回滚；主干永远可打包可运行。

## 0.6 风险清单（早期就要管理）

- **范围失控**：引擎自研与首作内容制作都很重；必须用“分阶段里程碑 + 可裁剪路线”控制。
- **AI 合规/版权/数据来源**：训练/微调/数据集必须可追溯；建议把“生成”定位为生产工具而非运行时依赖。
- **超大世界 + 百年时间线的复杂性**：需要事件化与分层 LOD 的世界状态表达，否则存档/任务/性能都会爆炸。
- **渲染与资产系统耦合过深**：会导致任何新技术试验都变成大手术；必须以 Render Graph + 资源虚拟化做解耦。
